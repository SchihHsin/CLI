# competitive-analysis Skill 使用说明

作者：时昕昱

---

## 这是什么

`competitive-analysis` 是一个竞品分析Skill，帮助你快速生成一份**完整的竞品分析 HTML 报告**，包含：

- 用户旅程地图
- 多产品触点对比表
- 每个触点的推荐设计方向
- 设计亮点提炼
- 前端开发规格文档

适用于任何数字产品领域（AI 工具、SaaS、电商、设计软件等）。

---

## 如何触发

1、随输入上传该文件；
2、上传到知识库中，在对话中说出以下类型的话，Skill 会自动激活：

```
帮我做 Claude Code 这类 AI CLI 工具的竞品分析
调研几款主流设计工具的输入交互方式
比较 Notion、Linear、Jira 的任务管理设计
```

**关键词**：竞品分析 / 调研竞品 / 比较产品 / 交互设计对比

---

## 开始前需要准备什么

Skill 会在开始前询问最多 3 个问题（如果你已经说清楚了就不问）：

| 问题 | 示例回答 |
|---|---|
| 分析什么产品领域？ | AI 命令行工具 |
| 分析目的是什么？ | 找设计差异化机会，给团队做汇报 |
| 有没有指定竞品？ | 不指定，让 AI 自己找主流产品 |

> 如果你需求模糊也没关系，Skill 会先生成第一版，你看了再反馈。

---

## Skill 会做什么（5 个阶段）

### Phase 1 · 自动搜集产品信息
AI 自主搜索，整理 4-6 个竞品的定位、界面特征、设计亮点。你不需要操作。

### Phase 2 · 生成用户旅程地图
绘制完整使用周期的旅程图，标注情绪曲线、摩擦点和设计机会。

### Phase 3 · 触点对比分析
每个交互触点独立一页，对比各产品的设计决策，附带 Terminal Mockup 或界面示意图。

### Phase 4 · 设计亮点提炼
跨触点综合，提炼 6-10 个最有价值的设计洞察。

### Phase 5 · 开发规格文档
为关键组件生成 TypeScript 接口定义、状态机、事件规格，开发可直接使用。

## 输出结果

Skill 会生成一组 HTML 文件，放在 `report/` 目录下：

```
report/
├── index.html           # 封面 + 用户旅程地图
├── shared.css           # 共享样式
├── tp-startup.html      # 触点分析：启动阶段
├── tp-intent.html       # 触点分析：意图表达
├── tp-execute.html      # 触点分析：执行阶段
├── ...                  # 其他触点页
└── sharing-slides.html  # 经验分享（可选）
```

用浏览器打开 `index.html` 即可查看，支持键盘方向键和滚动导航。

## 常见使用场景

**场景 1：快速竞品调研（5-8 个触点）**
```
帮我快速对比一下 Figma、Sketch、Framer 的组件管理交互，
重点看创建、搜索、复用这三个环节
```

**场景 2：完整 UX 研究（15-25 个触点）**
```
做一份完整的 AI CLI 工具竞品分析，
涵盖 Claude Code、Warp、Gemini CLI、Aider、Codex CLI，
覆盖从启动到收尾的完整用户旅程
```

**场景 3：设计决策支撑**
```
我们在设计新的代码审查界面，帮我看看 GitHub、GitLab、Linear
在 code review 触点上各自怎么做的，我需要做设计决策
```

## 使用建议

- **给"为什么"比给"怎么做"更有价值**：告诉 AI 你的判断标准，它会举一反三处理所有同类问题
- **看到第一版就给反馈**：不要等到全部完成，边生成边调整效率最高
- **触点维度可以拆分**：如果某个触点涵盖多个不同维度（比如"位置"和"编辑能力"），告诉 AI 拆成两个触点分析

---

## 技术说明

- 报告使用纯 HTML + CSS，无需框架，浏览器直接打开
- 滚动采用 `scroll-snap` 全屏楼层设计，每屏一个分析单元
- 图标全部使用 SVG 线性图标（无 emoji）
- 配色使用固定设计 Token，视觉风格统一
