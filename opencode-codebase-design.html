00  命令架构
单一命令 /codebase 配合子命令/选项，对标 opencode 的 Command.register 单命令架构。
命令功能说明
/codebase查看索引状态（默认）不带参数，简要展示当前三态之一
/codebase/init初始化索引本地代码上传云端并向量化（上传过程不单独呈现）
/codebase/detail查看状态详情进度百分比、索引时间、统计信息
/codebase/update更新索引（增量）仅对变更文件增量更新
/codebase/delete删除索引释放云端配额
/codebase/help查看帮助列出所有子命令
三种索引状态（不呈现上传中状态）：未构建索引 → 索引构建中（显示进度%）→ 索引构建完成。上传过程作为索引构建的前置步骤，对用户透明，仅感知这三态。

10  CLI 与 TUI 交互差异
交互形态进度展示错误提示用户操作
CLI（纯命令行） 文本进度条：进度：45% 纯文本：错误：[内容]。建议：[内容]。 键盘输入 Y / N / Ctrl+C
TUI（终端 UI，本方案） 右侧栏三态 + 消息区进度条 左线 alert（warning 橙），多行详情 键盘输入，可交互式处理
上方所有 mockup 为 TUI 形态。纯 CLI 形态下无右侧栏、无 alert 框，所有提示降级为单行纯文本（如 进度：45%、错误：网络连接失败。建议：请检查网络连接或稍后重试。）。
11  状态机 — 三态流转
// 索引状态（不对用户呈现「上传中」） type IndexState = "none" | "building" | "done" // 状态流转 none ──[首次打开 / init]──▶ building ──[完成]──▶ done done ──[update 有变更]─────▶ building ──[完成]──▶ done done ──[delete 确认]───────▶ none // building 状态展示进度，done 状态展示索引时间 { "state": "building", "progress": 30, // 仅 building 时有效，反映索引进度（非上传进度） "indexedAt": null, // done 时填充时间戳 "quota": { "used": 1, "total": 5 } } // 异常码（云端返回） QUOTA_INSUFFICIENT ──▶ warning 橙：配额超限 NETWORK_TIMEOUT ──▶ warning 橙：网络连接失败 INDEX_FAILED ──▶ warning 橙：索引构建失败
颜色语义：构建中 = 主题紫呼吸灯 #B494F5（进行中/活动）· 构建完成 = 绿色 #5BA854（稳定常亮）· 未构建 = 灰色（静默）。所有异常均为非致命问题，统一用 warning 橙 #FF8800，不使用红色，避免过度告警。
关于动效的实现（CLI 渲染方式）：终端无法做 GUI 的透明度渐变或填充补间，所有"动效"本质是字符逐帧重绘。本文档的动效均按真实 CLI 方式实现——
· Spinner：Braille 帧序列 ⠋⠙⠹⠸⠼⠴⠦⠧⠇⠏ 循环切换（~11fps），与 opencode / Claude Code 实际所用一致。
· 呼吸灯：圆点字符在 ● ◉ ◎ ○ 之间逐帧切换并回弹，用字符"亮度"模拟明暗，而非 opacity。
· 进度条：█ / ░ 字符按帧重排，填充边缘在 ▏▎▍▌▋▊▉ 子格字符间切换，模拟终端做平滑进度的 sub-cell 重绘手法。
