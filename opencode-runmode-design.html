07  三种运行模式说明
模式颜色标识行为适用场景
🛡 Sandbox 沙箱 黄色 仅白名单命令自动执行，非白名单拦截并 warning；网络访问按策略限制 生产环境、安全敏感项目、不熟悉的代码库
⚡ Auto 自动 绿色 AI 生成的所有命令自动执行，不做拦截 开发阶段、个人项目、信任环境
✋ Manual 手动 蓝色 每条命令执行前弹出确认，用户逐条审批 初次接触项目、危险操作调试
08  交互规格说明
场景行为键位 / 操作
/RunMode 触发 弹出三选一面板，当前模式置灰 ↑↓ ↵ Esc
选择 Sandbox Enter 进入沙箱配置编辑弹框 ↵ 进入编辑
沙箱编辑弹框 多行输入白名单命令 + 单选网络策略 + 确认/取消 Tab 切换区域 · ↑↓ 策略选项 · ↵ 确认 · Esc 取消
选择 Auto / Manual 直接切换，绿色通知 ↵ 确认
底栏模式标签 鼠标点击弹出快捷浮窗，支持三种模式快捷切换 鼠标点击 · Esc / 点击外部 关闭
沙箱拦截命令 非白名单命令不执行，消息区输出黄色 warning alert —
任意步骤取消 Esc 退出，不修改配置，面板/弹框消失 Esc
09  数据模型 — 配置字段
// opencode.json { "runMode": { "current": "sandbox", // "sandbox" | "auto" | "manual" "sandbox": { "allowedCommands": [ // 白名单命令列表 "git status", "git diff", "npm test", "npm run build", "go test ./..." ], "networkPolicy": "all", // "all" | "local" | "none" | "custom" "customPolicy": {} // 仅 networkPolicy="custom" 时生效，含 filesystem/network/resources } } } // 运行时判断 function canExecute(cmd) { if (runMode.current === "auto") return true if (runMode.current === "manual") return promptUser(cmd) if (runMode.current === "sandbox") { return runMode.sandbox.allowedCommands.some( allowed => cmd.startsWith(allowed) ) } }
