固定三层 · Status Bar + 输出流 + 输入区
全局架构 · Foundation 三层布局 — Status Bar + 输出流 + 固定输入区
设计说明
物理分离的三层结构
HAI CLI 将屏幕划分为三个职责明确的层：顶部固定 Status Bar 始终传达"我在哪里"；中间滚动输出流展示"发生了什么"；底部固定输入区提供"我能做什么"。三层位置永远不变，消除视线跳跃。
核心设计点
Status Bar（固定顶部） — 多 Session 标签 + 当前路径 + Stage，始终可见，不随输出滚动消失
输出流（可滚动） — 工具调用 Block + 进度 + 结果，写入操作加 badge，读取自动折叠
输入区（固定底部） — Context Chips + 主输入框 + 模式/费用/语音，永远在底部 20%
与竞品对比
输入位置状态栏输出结构
Claude Code固定底部无线性文字流
Warp底部持久状态栏Block 单元
Gemini CLI随输出滚动状态嵌入行内线性对话流
HAI固定底部固定顶部Block + 折叠
BEFOREOpenCode 现状
输入框跟随输出流末尾滚动，Agent 执行时无法输入；无全局状态栏；错误/进度信息与普通输出视觉相同
AFTERHAI 三层设计
输入框永远在底部，可随时输入；Status Bar 始终显示会话状态；输出流层次分明，写入操作即时可辨

Zone C · Input Area 固定底部全能面板 — Context Chips · 语音 · 费用角标
设计说明
三行结构 · 固定底部 20%
输入区由上至下分三行：Context Chips 行（已引用文件 + 上下文用量 + 费用）、主输入行（提示符 + 文本 + 语音）、工具行（模式切换 + 快捷键）。物理位置永远固定，消除"输入框跑走了"的焦虑。
组件详解
Context Chips
@file 引用以 chip 形式展示，点 × 移除；超过 3 个折叠为 "+N"；上下文用量 Chip 颜色随百分比变化（绿/黄/红）
费用角标
黄色累计费用常驻右侧，实时更新；点击展开按任务归因的明细列表；不出现在每条输出中（不切碎内容）
语音输入（Space）
长按 Space 激活；光标位置自动成为分析上下文；松开后三种模式：写入 / 发送 / 浮动编辑面板
模式切换（Cmd+I）
Auto（自主执行）/ Plan（提案后确认）/ Manual（逐步确认）；边框颜色随模式改变：Auto=蓝 / Plan=橙 / Manual=灰
BEFOREOpenCode
纯文本输入框，无 @file 结构化引用，无费用感知，无模式指示，无语音
AFTERHAI 输入区
Context Chips 管理上下文范围；费用角标实时可见；Space 语音填入；三种模式边框区分

HAI CLI · 输出流 · 设计点 5/12 工具调用块 — 读取折叠，写入高亮
设计原则
信噪比优先的输出流
AI 执行任务时会产生大量中间步骤。关键是：让重要信息突出，让噪声退后。
READ OPS — 自动折叠
读取类操作（Read / Search / List）自动折叠为计数徽章，不占用视觉空间。点击展开查看完整列表。
WRITE OPS — 展开 + 橙色 badge
写入类操作（Edit / Write / Delete）展开显示，加橙色边框 + WRITE badge，变更前需确认。
THINKING — 默认一行摘要
AI 推理过程折叠为单行摘要，点击展开完整思考链，不污染主输出流。
EXEC OPS — 红色 badge
执行命令（Bash / Run）加红色边框，始终需要用户确认，不可自动执行。
BEFOREOpenCode
所有 Tool Call 平铺输出
JSON + 换行，无区分
Read 和 Write 混在一起
AFTERHAI 输出流
READ 折叠计数 · WRITE 橙色展开 · THINK 单行摘要 · EXEC 红色必确认

HAI CLI · 执行流 · 设计点 6/12 五状态模型 — 任务全生命周期可见
核心设计
用户永远知道 AI 在哪一步
HAI 把每个任务分解为 5 个状态，Status Bar 右侧始终显示当前状态，不会"黑屏执行"。
S1
Initial — 就绪
欢迎画面 · 自动加载工作区记忆 · 显示上次任务摘要
S2
Working — 执行中
步骤计数 + 用时 + Esc 浮动按钮 · 可插话追加指令
S3
Confirmation — 等待确认
颜色编码风险 · 三档授权 · 标签 ⚠ 闪烁 · 手机推送
S4
Preview — 结果预览
默认摘要行 · 点击展开完整 diff · ≥3 文件触发 Review Panel
S5
Completion — 完成
验收摘要 · 存入记忆 · 可直接输入下一目标
BEFORE传统 CLI
命令执行 = 黑盒
不知道执行到哪一步
不知道需要多久

Status Bar 右侧始终显示 Stage · 任务状态完全可见

HAI CLI · 权限确认 · 设计点 7/12 颜色编码风险 — 三档授权 + 修改后执行
设计原则
让用户感知风险等级，而不是盲目确认
权限确认必须传递风险信息。颜色系统让用户一眼判断操作危险程度。
蓝色 — 读取操作
Read / Search / List 文件系统，低风险，可快速批准

Edit / Write / Delete 文件，中等风险，建议查看 diff
红色 — 执行命令
Bash / Run / Deploy，高风险，必须显式确认
三档授权粒度
仅此次本次操作有效，下次重新确认
会话级本次 session 内自动批准同类操作
永久级写入 settings.json，后续不再询问
修改执行打开编辑器修改命令参数后再执行
BEFOREClaude Code
二选一：Allow / Deny
不传递风险等级
无"修改后执行"选项
AFTERHAI 权限确认
颜色编码风险 · 三档授权 · 第四选项"修改后执行"

HAI CLI · 错误处理 · 设计点 11/12 错误分类 — 自动重试 · 透明上报 · 升级路径
设计原则
错误是信息，不是墙
HAI 对错误进行分类，根据类型决定是自动重试、上报给用户还是升级到人工介入。
可恢复错误 — 自动重试
网络超时 / API 限流 / 临时失败 → 自动重试 3 次，透明显示重试进度
逻辑错误 — 上报用户
format 不兼容 / UB 溢出 / shape 推导失败 → 具体描述 + 建议方案，等待用户决策
严重错误 — 升级路径
编译器崩溃 / 硬件异常 / 未知错误 → 保存现场 + 提示联系支持 + 提供诊断包
错误上报规范
1. 明确错误类型（非错误码）
2. 当前状态 + 已完成步骤
3. 可选操作：重试 / 修改参数 / 跳过 / 中止
BEFORE错误 = 崩溃
任意错误中止任务
无状态保存
需要重新开始
AFTERHAI 三级错误流
自动重试 · 类型化上报 · 升级路径 · 现场保存

退出时自动弹出会话摘要卡，让用户对本次 AI 协作有完整认知，支持复制分享。
摘要卡包含内容
📋 完成任务数 + 类型归因（优化/修复/新增）
📁 变更文件数 + 总增删行数
🔤 Token 用量（输入 + 输出 + 缓存）
💰 本次会话总费用（含明细）
⏱ 总用时 + 各任务耗时分布
💾 已存入记忆的内容摘要
费用角标常驻
费用角标常驻在 Chips 区右侧，实时更新，不在每条输出中重复显示。退出时在摘要卡中展示完整明细。
