在主任务进行中插入侧链问答——答案以临时浮层呈现，问答双向不进入对话历史，上下文窗口零污染。

ephemeral agent inherits full context no tools won't enter history
1
主任务进行中，用户输入 /btw

Claude 正在执行多文件重构任务。底部输入框中输入 /btw 后，自动补全提示命令用途。

主任务不会中断，状态栏持续滚动。

>/btw
/btw <question> — ask a side question, no tools, won't enter history
2
提交问题 → fork 上下文，派生临时 agent

回车后系统 fork 当前会话上下文，派生一个 ephemeral agent：

继承全部已读代码与历史决策，但禁用所有工具调用。浮层出现，主任务继续在后台运行。

claude — ~/project
●Refactoring auth module...still running
>/btw what was that config file we read earlier called?

ephemeral no tools won't enter history
3
答案在浮层中呈现

侧链 agent 基于已有上下文回答，结果渲染在独立浮层中，与主任务流视觉隔离。

主任务始终未受影响，继续推进。按 esc 可随时关闭浮层。

4
工具被请求 → 客户端拒绝态

若侧链问题隐含需要工具（如"读一下那个文件"），客户端层硬拦截 tool_use。

Agent 基于已有上下文回答，并提示改用主对话或 subagent。浮层变为琥珀警告色。

claude — ~/project
>/btw open routes.ts and tell me line 40
◆  Side question esc to dismiss
⚠ Side questions cannot use tools
I can't read files here. From what's already in context, routes.ts handles the login redirect. To read it live, ask in the main thread or use a subagent.
5
关闭浮层 → 历史保持干净

按 esc 关闭浮层。问答双向都不入栈——既不进 history，也不进 context window，完全 ephemeral。

主对话从中断处无缝继续，终端底部提示零 token 被写入历史。
