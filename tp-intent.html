Touchpoint 02A · Our Direction输入框位置 & 形态 — 我们的方向
固定底部为基础，长任务自动提示分离视图
默认固定底部（成本最低、体验最佳的起点）。当 AI 进入超过 5 步的长任务执行时，输入框上方出现"切换到 Agent 视图？"提示，用户可选择切换或留在当前视图，不强制。
01
输入框锚定底部，始终在视野内
无论对话有多长，输入框永远在屏幕底部。用户不会"找不到输入框"——这是减少使用摩擦最有效的单一设计决策。固定底部同时让输出区域可以自由滚动，历史内容随时可查。
02
长任务自动提示切换分离视图
检测到 AI 开始执行超过 5 步的长任务时，在输入框上方出现轻量提示："任务正在后台执行（3/8 步）· 切换到 Agent 视图？"。用户可切换或忽略，两种视图都完整支持。
03
分离视图：左历史右任务，可同时操作
切换到分离视图后，左侧为历史对话（可继续输入短任务），右侧为当前 Agent 任务状态。用户在等待 Agent 执行时可以同时做其他短任务，不需要干等。

Touchpoint 02B · Our Direction输入框编辑能力 — 我们的方向
多行编辑 + Ctrl+R 历史搜索 + 语法高亮
多行编辑让复杂指令可以换行写清楚；Ctrl+R 跨会话历史搜索让常用指令可以快速复用；输入区语法高亮（@ 引用蓝色 / /命令紫色 / 路径绿色）让指令结构一目了然。
01
Shift+Enter 换行，光标自由移动
支持多行编辑，Shift+Enter 换行，Enter 提交。光标可自由定位，Ctrl+A 全选，Ctrl+K 删到行末。让用户写复杂多步指令时，体验与在编辑器里写注释一样。
02
Ctrl+R 历史搜索，Tab 编辑后复用
Ctrl+R 触发历史搜索弹窗，支持模糊搜索，结果显示完整指令预览。Enter 直接复用，Tab 填入输入框进入编辑状态，让用户可以在历史基础上微调再复用。跨会话持久 200 条。
03
输入区语法高亮，结构一目了然
@引用蓝色，/命令紫色，文件路径绿色，普通文字白色。这个高亮不只是好看——它给用户"我的指令被正确解析了"的信心，同时让错误（比如 @ 后面打错文件名）在提交前就能发现。

Touchpoint 03 · Our Direction模式指示符 — 我们的方向
Badge + 边框颜色双重强调，Cmd+I 显式切换
Agent Mode 紫色边框 + Badge，Terminal Mode 灰色边框 + Badge。两种模式视觉差异足够大，用户扫一眼就能识别。Cmd+I 显式切换，不做隐式自动检测——避免误判带来的信任损耗。
01
两种模式视觉差异要足够大
Agent Mode 和 Terminal Mode 在颜色（紫色/灰色）、边框粗细、Badge 图标（✦/⬣）三个维度都有明确差异。不能只靠一个小文字标注——那样用户很容易忽视，在错误模式下操作。
02
模式状态作为输入框的一部分，始终可见
模式指示不放在 Banner（会滚走），不放在状态栏（离输入框太远）。而是作为输入框顶部的 Badge 行，用户在输入的同时就能看到当前模式，不需要移开视线。
03
Cmd+I 显式切换，快捷键提示常显
切换快捷键 Cmd+I 显示在 Badge 行旁边（小灰字），发现成本为零。切换时边框颜色平滑过渡（200ms 动画），让用户感知到模式确实发生了变化，而不是静默切换。

Touchpoint 04 · Our Direction添加上下文 — 我们的方向

Touchpoint 05 · Our DirectionContext Chips — 我们的方向
渐进式 Chips：初始最少，随使用深度丰富
Chips 不是越多越好。初始状态只显示两个最关键的：权限模式（影响自动化安全性）和费用角标（影响使用决策）。引用文件后才出现文件 Chip。Git 状态等信息按需展示，不强迫。
01
权限模式 Chip 最重要，颜色区分
权限模式是所有状态中最需要始终可见的——用户必须随时知道 AI 会不会自动执行危险操作。Normal（绿色）、Auto-accept（橙色）、Plan Mode（蓝色），颜色区分，一眼可识，永远不会漏掉。
02
费用角标实时更新，点击展开明细
累计费用以黄色小角标显示在 Chips 区右侧，随每次操作实时更新。不出现在每条输出中（避免切碎内容），但始终在视野里（避免被遗忘）。点击展开按任务归因的费用明细。
03
超 3 个文件折叠，保证输入空间
引用文件 Chip 超过 3 个时，自动折叠为 "+N 更多"。点击展开查看完整列表，点击 × 移除。这个约束保证输入框永远有足够的输入空间，不会被 Chips 占满导致打字区域过窄。

Touchpoint 06 · Our Direction模型选择器 — 我们的方向
Chip 内嵌下拉 + 三维标注 + 子 Agent 自动降级
模型以 Chip 内嵌在输入框，点击弹出下拉列表，每个模型标注成本/速度/最适场景三个维度，让用户不查文档就能做出合理选择。当前模型名始终在 Chip 中可见，不会迷失。
01
三维标注：成本 / 速度 / 最适场景
下拉列表中每个模型显示：成本（$/$$/$$$）、速度（极快/快/慢）、最适场景（通用/编码/深度分析）。让"写单元测试选 Haiku（$ · 极快 · 通用）"这样的判断可以直接在选择器内完成。
02
当前模型名 Chip 始终可见
当前使用的模型以 Chip 形式持续显示在输入框，不需要滚回 Banner 或执行 /model 命令才能确认当前是什么模型。这是最基础的状态透明度，不应该缺失。
03
子 Agent 自动使用低成本模型
多 Agent 场景中，子 Agent（读文件、搜索、探索性任务）自动使用 Haiku 等低成本快速模型执行，主 Agent 继续使用用户选择的模型。在任务面板中显示各 Agent 使用的模型和产生的费用。

Touchpoint 07 · Our Direction历史记录 — 我们的方向
Ctrl+R 模糊搜索 + 完整预览 + Tab 编辑后复用
历史跨会话持久化 200 条，Ctrl+R 触发模糊搜索，结果显示完整指令内容预览（不只是截断的前几个字）。Tab 键填入输入框进入编辑状态，让用户在历史基础上微调再复用。
01
Ctrl+R 搜索，结果显示完整预览
搜索时，结果项显示指令的完整内容预览（多行折叠为前 2 行）。用户能看到这条历史究竟写了什么，再决定是否复用。大幅减少"复用了错误历史"的情况。
02
Enter 复用，Tab 编辑后复用
Enter 直接复用历史指令，Tab 把历史内容填入输入框进入编辑状态。后者让用户可以在历史基础上微调（换一个文件名、改一个参数），而不是只能原样复用。这个细节大幅提升历史复用率。
03
按项目分组，跨会话持久 200 条
历史记录按工作目录/项目分组，防止不同项目的历史混在一起。跨会话持久化最近 200 条，超出时按时间淘汰最旧的。会话结束时自动保存，不需要用户手动操作。
