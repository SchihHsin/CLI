# CLI Analysis Report — 核心结论速查

> 源自 `cli-analysis-report.html`，系统对比 Claude Code / Warp / Gemini CLI / Aider / Codex CLI。

---

## 一、大背景

- **CLI vs MCP**：75 项测试 CLI 全胜，Token 成本低 10–32 倍，可靠性 ≈100% vs MCP 的 72%
- Perplexity 2026.03 宣布放弃 MCP，全面转向 REST API + CLI
- JetBrains 2026：Claude Code 工作使用率 18%，一年涨 6 倍

---

## 二、页面框架推荐

**结论：固定分离式 + 轻量 Block**

| 方案 | 谁在用 | 优势 | 局限 |
|---|---|---|---|
| 固定底部输入 + 输出流 | Claude Code | 输入稳定，IDE 级编辑 | 无全局状态栏，SSH/tmux 差 |
| Block + 状态栏 | Warp | 状态最完整，Block 可操作 | GPU 渲染，SSH 不兼容 |
| 线性对话 + PTY | Gemini | 1M 上下文，PTY 跑任意工具 | 无固定输入框 |

**我们的方向：**
- 固定底部编辑器（实现成本最低）
- 轻量 Block：**只对写入操作**加可折叠 badge，不依赖 GPU
- 输入框内嵌 Context Chips + 费用角标

## 三、18 条核心设计原则

### PHASE 01 — 启动
1. Banner 只保留**模型名 + 权限模式**两个信号，其余收进 Chips 按需查阅
2. 新用户首次触发一次性引导 overlay（三个核心操作：@ 引用 / /plan 提案 / Esc 中断）

### PHASE 02 — 执行中
3. 模式 Badge 颜色区分（Agent 蓝色 / Terminal 灰色），永远可见
4. @ 补全延迟 <100ms，Chips 超 3 个折叠为 "+N"
5. 读取类 Tool Call **自动折叠**计数；写入类**展开**加 Block badge
6. Thinking 默认**一行摘要可展开**，不污染输出流
7. 进度：**步骤计数 + 用时 + Esc 浮动按钮**始终可见

### PHASE 03-06 — 确认 / 审查 / 迭代 / 收尾
8. 权限确认**颜色编码风险**：蓝=读取 / 橙=写入 / 红=执行命令
9. 三档授权（仅此次 / 会话 / 永久）+ **"修改后执行"第四选项**
10. 默认输出**摘要行**（✔ 3 files · +18 -6 · $0.023），点击展开完整 diff
11. 变更 ≥ 3 文件时自动建议 Review Panel（可忽略，不强制）
12. Accept all 主路径 + By file 次要路径 + **Ask AI 追问始终内嵌**
13. **费用角标常驻**输入框 Chip 区，实时更新，不出现在每条输出中
14. Agent 等待时**橙色暂停标记 + 桌面通知**，不卡死其他任务
15. Esc 按钮**始终浮动**在输入框旁（红色），中断后输入框立刻可用
16. 上下文用量**仪表盘常驻** Chip 区（<70% 灰 / 70-90% 黄 / >90% 红）
17. /fork 创建快照分支（原分支保留）；/compact 轻量压缩；/clear 全清
18. 退出时**自动展示会话摘要卡**（任务数 / 文件数 / Token / 费用 / 用时）

### 总体原则
> 让「AI 在做什么」始终可见，让「用户能做什么」始终触手可及。  
> 透明度不是噪音，而是信任的基础。自动化程度不是目标，**掌控感才是**。

## 五、多 Agent 架构

- **起步**：Hub & Spoke（主 Agent 委派，子 Agent 独立上下文，只返回摘要）
- **演进**：P2P Mailbox（Agent Teams，实验性）
- **等待处理**：橙色暂停标记 + 通知，不阻塞其他任务（Warp 的做法最安全）
- **子 Agent 隔离**：独立上下文执行，完成后只向主 Agent 返回摘要，控制 Token 成本

---

## 六、OpenCode 的 6 大设计局限（需填补）

1. 启动态：无权限模式 / 工具状态显示
2. 输入区：无结构化上下文注入（@file 等）
3. 执行过程：Tool Call 纯文本流，无折叠，无进度
4. 变更审查：原始 diff，无 Accept/Reject 工作流
5. 视觉系统：全局单色，错误与普通输出无区分
6. 多 Agent：无统一管理面板，子任务不可见
