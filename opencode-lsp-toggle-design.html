06  状态机 — 配置字段与判断逻辑
// opencode.json { "lsp": { "enabled": false, // 当前运行时生效值（重启后读取） "pendingEnabled": false // 用户设置的目标值 } } // 执行 /lsp-toggle 时的锁定判断 const isLocked = lsp.enabled !== lsp.pendingEnabled if (isLocked) return showErrorAlert() // 不弹面板，直接报错 // 用户确认后写入 lsp.pendingEnabled = params.enable // 重启后执行（两值同步 → 解除锁定） lsp.enabled = lsp.pendingEnabled

08  交互规格说明
场景行为键位
/lsp-toggle（正常） 弹出选择面板，当前状态置灰，对侧橙色实心高亮 ↑↓ ↵ Esc
关→开：选择「开启」 面板切换为黄色左线预警 + 二次确认，默认高亮「确认开启」 ↵ 直接确认 · Esc 取消
开→关：选择「关闭」 无二次确认，直接写入 pendingEnabled = false —
操作完成 绿色左线通知；Sidebar 变黄；status bar 出现 r 立即重启 r 立即重启
/lsp-toggle（锁定） 不弹面板；蓝色 info 显示上次记录；红色 error 说明锁定原因 r 立即重启解锁
任意步骤取消 Esc 退出，不修改任何配置，面板消失 Esc
