01　文本即数据

整个字段就是
一段可编辑的 CSS 值

不再把数字和单位拆成两个控件。整个字段是一个文本输入，你可以三击选中再输入 1.5rem，或者只输入 120 保留当前单位 —— 完全是 Figma / DevTools 的成熟范式。

解析在 Enter 或失焦时发生。支持 px / % / em / rem / vh / vw / 关键字 / calc(...)。无效输入边框变红，但不阻止继续编辑。

02　浮层而非内嵌

单位选择器是悬浮的，
不占布局空间

下拉打开时不会把下面的属性挤下去。它是一个 position: fixed 的浮层，带连接箭头指向触发它的字段，自动避让屏幕边界。

这样设计有两个好处：一是面板布局稳定，眼睛不必重新定位；二是浮层可以做得更宽更舒展，承载单位说明、等效值预览、换算策略切换等丰富信息。

03　两种入口共存

键盘直达，
鼠标辅助发现

熟练用户用键盘：选中输入新值，一气呵成。这是主要交互。

新手或忘记单位名时，鼠标悬停字段右侧出现 ▾，点击打开浮层，里面有所有单位的中文说明、当前值在新单位下的等效预览，以及"保视觉/保数值"的换算策略 —— 一个完整的发现入口。
