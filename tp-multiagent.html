Design Principles · Summary18 条核心设计原则
PHASE 01-02 · 启动 & 表达意图
零配置首次体验，30 秒内开始第一个任务
Banner 只展示：模型名 + 权限模式两个关键信息
新用户首次触发一次性引导，此后不再重复
输入框固定底部，首次加载即获焦点
模式 Badge 颜色区分（Agent 紫色 / Terminal 灰色），永远可见
@ 补全延迟 <100ms，Chips 超 3 个折叠为 "+N"
PHASE 03 · 等待执行（情绪最低谷）
Thinking 默认一行摘要，点击展开，完成后自动折叠
读取类 Tool call 自动折叠为计数，写入类展开并标色
高危操作自动触发 Plan Mode 先提案再执行
步骤计数（n/N）+ 整体进度条，双重反馈
失败高亮红色 + "让 AI 修复"快捷入口内嵌进度视图
Esc 中断按钮始终浮动可见，不需记快捷键
PHASE 03-06 · 确认 / 审查 / 迭代 / 收尾
弹窗颜色编码风险等级（蓝色读取 / 橙色写入 / 红色执行）
三档授权（仅此次/会话/永久）+ "修改后执行"第四选项
默认输出摘要行，变更 ≥ 3 文件自动建议 Review Panel
输出末尾恒有追问入口（✦），驱动自然迭代节奏
费用角标始终在 Chip 区可见，点击展开按任务明细
退出时展示会话摘要卡，强化完成感和费用意识
总体原则：让"AI 在做什么"始终可见，让"用户能做什么"始终触手可及。透明度不是噪音，而是信任的基础。自动化程度不是目标，掌控感才是。每一个设计决策都应该问：「这让用户更有掌控感了吗？」
